## Что умеет Luma

- 🎤 Принимает **голосовые** и текстовые сообщения в Telegram
- 💬 Принимает текстовые сообщения в MAX
- 📅 Создаёт события в Google Calendar (распознаёт дату/время из текста)
- ⏰ Присылает напоминания: за 1 час, за 15 мин, за 5 мин
- 🌅 Утренний обзор дня в 8:00, 12:00 и 18:00
- 🖥️ **Веб-дашборд** — управление ботами, просмотр событий, логи
- 🔔 **Системный трей** — иконка в области уведомлений Windows
- 🔄 **Watchdog** — авто-перезапуск ботов при падении

## Быстрый старт

```powershell
# Активировать виртуальное окружение
.\Activate.ps1

# Установить зависимости
pip install -r requirements.txt

# Запустить приложение (с консолью)
python -m app.main

# Запустить приложение (в фоне, без консоли)
pythonw -m app.main
```

После запуска откроется браузер с дашбордом: **`http://localhost:52025`**

Также можно запустить двойным кликом по ярлыку **Luma2025** на рабочем столе.

## Автозапуск с Windows

Приложение само стартует при входе в систему. Файл `Luma2025_startup.bat` находится в папке автозагрузки Windows.

Чтобы отключить: удали файл из папки `shell:startup` (Win+R → `shell:startup`).

## Распространение (как передать другому)

Если хочешь поделиться приложением с коллегой или установить на другой компьютер:

### Что отправлять

Собери актуальный EXE и заархивируй папку `dist/Luma2025/` без секретов:

```powershell
# 1. Пересобрать EXE (если были изменения в коде)
pyinstaller Luma2025.spec --noconfirm

# 2. Создать архив БЕЗ секретных файлов
#    (взять только папку dist/Luma2025/)
```

**НИКОГДА не отправляй эти файлы — в них твои личные ключи и токены:**

- `.env`
- `credentials.json`, `max_credentials.json`
- `token.json`, `max_token.json`
- `*.key`, `*.pfx`

### Что нужно получателю

1. Распаковать архив в любую папку
2. Создать файл `.env` со **своими** токенами (см. [Настройка с нуля](#настройка-с-нуля))
3. Получить `credentials.json` в [Google Cloud Console](https://console.cloud.google.com/apis/credentials)
4. Запустить `Luma2025.exe`
5. При первом запуске откроется браузер — авторизоваться в Google
6. Готово — боты работают, дашборд на `http://localhost:52025`

## Бэкапы

### Создать бэкап

```powershell
python backup_project.py
```

Или кнопка **📦 Бекап** в дашборде. Архив (~3 МБ) сохраняется в `backups/`. Хранится до 5 копий.

В архив **не попадают**:

- `venv/`, `dist/`, `build/`, `.git/`, `backups/`, `__pycache__/`
- `.env`, `credentials.json`, `token.json` (секреты)

### Восстановление

1. Распаковать архив в чистую папку
2. Скопировать `.env`, `credentials*.json`, `*_token.json` обратно
3. Установить зависимости:

```powershell
python -m venv venv
.\Activate.ps1
pip install -r requirements.txt
```

## Настройка с нуля

1. **Python 3.10+** — установить
1. **FFmpeg** — скачать с [ffmpeg.org](https://ffmpeg.org), указать путь в `.env`
1. **Telegram бот** — создать через [@BotFather](https://t.me/BotFather), получить токен
1. **MAX бот** — получить токен из настроек канала в MAX
1. **Google Calendar API** — создать проект в Google Cloud Console, скачать `credentials.json`
1. **Опубликовать OAuth** — чтобы окно входа не выскакивало каждые 7 дней.
   Нужно сделать для обоих проектов (Telegram и MAX используют разные Google-аккаунты):

   **Telegram:**
   - Открыть [Google Cloud — Telegram](https://console.cloud.google.com/apis/credentials/consent?project=460773708593)
   - Слева вкладка **«Аудитория»** → нажать **«Опубликовать приложение»**

   **MAX:**
   - Открыть [Google Cloud — MAX](https://console.cloud.google.com/apis/credentials/consent?project=752409691648)
   - Слева вкладка **«Аудитория»** → нажать **«Опубликовать приложение»**

   После публикации статус сменится на «В производстве» — токен будет жить постоянно.

   **Когда токен может стать недействительным:**

   1. **Первый запуск** — `token.json` ещё нет, нужно нажать «Переподключить» в Настройках
   2. **Режим «Тестирование»** — токен живёт 7 дней, потом требует повторного входа (решено публикацией в Production)
   3. **Смена пароля Google** или удаление доступа приложению в настройках аккаунта Google

   **Как восстановить:** дашборд → Настройки → кнопка «Переподключить» рядом с OAuth-статусом

1. Создать `.env` файл с токенами (см. пример ниже)

```env
# Telegram
TELEGRAM_BOT_TOKEN=ваш_токен
TELEGRAM_ENABLED=true
CHAT_ID=ваш_chat_id

# MAX
MAX_ENABLED=true
MAX_BOT_TOKEN=ваш_токен
MAX_OPERATOR_CHAT_ID=ваш_chat_id
MAX_API_BASE_URL=https://platform-api.max.ru

# Google Calendar
CREDENTIALS_FILE=credentials.json
TOKEN_FILE=token.json
MAX_CREDENTIALS_FILE=max_credentials.json
MAX_TOKEN_FILE=max_token.json
CALENDAR_ID=primary
MAX_CALENDAR_ID=primary

# FFmpeg
FFMPEG_PATH=C:\ffmpeg\bin\ffmpeg.exe

# Часовой пояс
USER_TIMEZONE=Europe/Moscow
```

1. Запустить:

```powershell
python -m app.main
```
